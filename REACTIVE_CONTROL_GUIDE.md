## 核心概念

### 什么是反应控制？

反应控制是一种**基于规则的控制系统**，通过显式的决策规则将感知信息映射到动作。

```
感知 → 特征提取 → 决策规则 → 动作
 ↓        ↓          ↓        ↓
状态    距离、角度   IF-THEN   速度、转向
      可见性、障碍   规则
```

### 反应控制 vs 学习控制

| 特性 | 反应控制 | 学习控制（ACT） |
|------|---------|-----------------|
| **实现方式** | 硬编码规则 | 神经网络学习 |
| **输入** | 计算特征 | 原始状态 + 图像 |
| **输出** | 动作 | 动作序列（Chunk） |
| **可解释性** | 高 | 低 |
| **性能** | 稳定、可预测 | 可能更优 |
| **开发时间** | 快 | 慢 |
| **泛化能力** | 有限 | 强 |

## 反应控制的逻辑

### 1. 感知层：状态表示

反应控制使用**计算特征**而非原始状态：

```typescript
// 原始状态
robotState = {x, z, rotation, velocity}
ballState = {x, z, velocity}

// 计算特征
distance = √((ball_x - cam_x)² + (ball_z - cam_z)²)
angleDiff = atan2(ball_x - cam_x, ball_z - cam_z) - rotation
isVisible = |angleDiff| ≤ FOV
isBlocked = raycast(cam_pos, ball_pos)
```

**关键特征：**
- **距离 (distance)**：决定前进速度
- **角度差 (angleDiff)**：决定转向速度
- **可见性 (isVisible)**：决定是否搜索
- **遮挡 (isBlocked)**：决定是否改道

### 2. 决策层：分层决策树

```
                    ┌─ 距离 ≤ 1.5m ─┐
                    │                │
            ┌─ 可见性 ─┤              ├─ 停止 (v=0, ω=0)
            │       │                │
开始 ──────┤       └─ 距离 > 1.5m ─┤
            │                        ├─ 靠近 (v>0, ω=angleDiff)
            │       ┌─ 角度 > 30° ──┤
            │       │                ├─ 减速靠近
            │       └─ 角度 ≤ 30° ──┤
            │                        └─ 全速靠近
            │
            └─ 不可见 ──────────────── 原地旋转 (v=0, ω>0)
```

### 3. 控制层：参数化控制

```typescript
// 转向控制（比例控制）
targetTurn = angleDiff * 1.0;
targetTurn = Math.max(-turnSpeed * 1.5, Math.min(turnSpeed * 1.5, targetTurn));

// 前进控制（距离相关）
targetSpeed = Math.min(speed, (distance - 1.5) * 0.5);

// 转向时减速
if (Math.abs(angleDiff) > Math.PI / 6) {
    targetSpeed *= 0.5;
}
```

### 4. 平滑层：动作平滑

```typescript
// 速度平滑（一阶低通滤波）
robotState.velocity = robotState.velocity * 0.5 + targetSpeed * 0.5;
robotState.angularVelocity = robotState.angularVelocity * 0.5 + targetTurn * 0.5;
```

## 反应控制与模型训练的关系

### 关键洞察

**反应控制教会我们：**
1. **哪些特征是关键的**
2. **如何组织这些特征**
3. **什么样的决策是有效的**
4. **如何评估模型性能**

### 映射关系

```
反应控制的特征          模型训练的状态向量
─────────────────────────────────────────
distance          ←→  state[9]
angleDiff         ←→  state[10]
isVisible         ←→  state[12]  ← 新增
isBlocked         ←→  state[13]  ← 新增
ballAccel         ←→  state[14-15] ← 新增
```

### 为什么这个映射很重要？

```
反应控制的逻辑：
  IF distance ≤ 1.5 AND isVisible:
      action = STOP
  ELIF NOT isVisible:
      action = SEARCH
  ELIF distance > 1.5 AND isVisible:
      action = APPROACH

模型应该学会：
  f(distance, angleDiff, isVisible, ...) → action

这正是反应控制所做的！
```

## 特征工程原理

### 1. 为什么需要特征工程？

```
学习效率对比：

方案 A（原始状态）：
  Epoch 1-10:   学习计算距离
  Epoch 11-20:  学习计算角度
  Epoch 21-30:  学习计算可见性
  Epoch 31-50:  学习根据特征做决策
  ────────────────────────────
  总需要：50 个 Epoch

方案 B（计算特征）：
  Epoch 1-20:   直接学习根据特征做决策
  ────────────────────────────
  总需要：20 个 Epoch

效率提升：2.5 倍！
```

### 2. 特征的三个层次

#### 第 1 层：原始状态（8维）
```
state[0-1]:   机器人位置 (x, z)
state[2-3]:   机器人朝向 (sin(θ), cos(θ))
state[4]:     机器人速度
state[5-6]:   小球位置 (ball_x, ball_z)
state[7-8]:   小球速度 (ball_vx, ball_vz)
```

#### 第 2 层：基础计算特征（4维）
```
state[9]:     距离 (distance)
state[10]:    相对角度 (angleDiff)
state[11]:    碰撞标志 (isColliding)
state[12]:    可见性 (isVisible)
```

#### 第 3 层：高级特征（4维）
```
state[13]:    被遮挡 (isBlocked)
state[14-15]: 小球加速度 (ballAccelX, ballAccelZ)
```

### 3. 特征的计算方法

#### 距离特征
```typescript
const dx = targetX - camX;
const dz = targetZ - camZ;
const distance = Math.sqrt(dx * dx + dz * dz);
```

#### 角度特征
```typescript
const angleToTarget = Math.atan2(dx, dz);
let angleDiff = angleToTarget - robotRotation;
// 归一化到 [-π, π]
while (angleDiff > Math.PI) angleDiff -= Math.PI * 2;
while (angleDiff < -Math.PI) angleDiff += Math.PI * 2;
```

#### 可见性特征
```typescript
const FOV = Math.PI / 4;  // 45°
const isVisible = Math.abs(angleDiff) <= FOV;
```

#### 遮挡特征（射线投射）
```typescript
let isBlocked = false;
if (isVisible && distance > 1.0) {
    const steps = Math.floor(distance / 0.5);
    for (let i = 1; i < steps; i++) {
        // 检查射线上的每个点是否与墙相交
        if (checkWallCollision(rayPoint)) {
            isBlocked = true;
            break;
        }
    }
}
```

#### 加速度特征
```typescript
const ballAccelX = (ballVelX - prevBallVelX) / dt;
const ballAccelZ = (ballVelZ - prevBallVelZ) / dt;
```

## 训练数据要求

### 1. 数据采集的分阶段策略

#### 第 1 阶段：基础能力（5-10 条轨迹）

**目标**：让模型学会基本的追踪

**配置**：
- 场景：基础场景，无障碍物
- 小球模式：固定航迹（圆形运动）
- 小球速度：0.3（慢速）
- 小球半径：5 米
- 采集时间：每条 30-45 秒

**数据特征**：
- 小球始终在视野内
- 距离变化：1.5m - 8m
- 角度变化：-45° - 45°
- 无遮挡情况

**模型应学会**：
- 根据距离调整前进速度
- 根据角度调整转向速度
- 保持稳定的追踪

#### 第 2 阶段：进阶能力（10-15 条轨迹）

**目标**：提高对不规则运动的适应能力

**配置**：
- 场景：基础场景，无障碍物
- 小球模式：混合运动（圆形 + 随机扰动）
- 小球速度：0.5-1.0（中等速度）
- 小球半径：3-8 米（变化）
- 采集时间：每条 45-60 秒

**数据特征**：
- 小球运动不规则
- 距离变化更剧烈
- 角度变化更频繁
- 偶尔移出视野

**模型应学会**：
- 预测小球的加速度
- 应对突然的方向改变
- 在小球移出视野时搜索

#### 第 3 阶段：高级能力（10-20 条轨迹）

**目标**：在复杂

**配置**：
- 场景：复杂场景，有障碍物
- 小球模式：混合运动
- 小球速度：1.0-1.5（快速）
- 小球半径：3-8 米（变化）
- 采集时间：每条 60-90 秒

**数据特征**：
- 小球被障碍物遮挡
- 需要绕过障碍物
- 高速运动
- 复杂的追踪轨迹

**模型应学会**：
- 在被遮挡时预测小球位置
- 规划绕过障碍物的路径
- 在高速运动中保持稳定

### 2. 数据质量要求

#### 覆盖率要求

```
距离覆盖：
  ✓ 近距离（1.5m - 2.5m）：20% 的数据
  ✓ 中距离（2.5m - 5m）：50% 的数据
  ✓ 远距离（5m - 8m）：30% 的数据

角度覆盖：
  ✓ 正前方（-15° - 15°）：30% 的数据
  ✓ 侧前方（-45° - 45°）：50% 的数据
  ✓ 侧方（-90° - 90°）：20% 的数据

可见性覆盖：
  ✓ 完全可见：70% 的数据
  ✓ 部分可见：20% 的数据
  ✓ 不可见：10% 的数据

遮挡覆盖：
  ✓ 无遮挡：80% 的数据
  ✓ 部分遮挡：15% 的数据
  ✓ 完全遮挡：5% 的数据
```

#### 动作多样性要求

```
前进动作：40%
  - 快速靠近
  - 缓慢靠近
  - 停止

转向动作：40%
  - 左转
  - 右转
  - 直行

搜索动作：20%
  - 原地旋转
  - 缓慢移动搜索
```

#### 碰撞避免要求

```
✓ 碰撞次数 < 2 次/分钟
✓ 碰撞后能快速恢复
✓ 避免重复碰撞同一位置
```

### 3. 数据采集的最佳实践

#### 采集前的准备

```
1. 调整小球速度
   - 从慢速开始（0.3）
   - 逐步增加到目标速度
   - 确保能跟上

2. 熟悉控制方式
   - 练习基本操作
   - 理解小车的运动特性
   - 找到最舒适的操作方式

3. 规划采集路线
   - 确定小球的运动范围
   - 避免频繁碰撞
   - 覆盖不同的场景
```

#### 采集过程中的注意事项

```
1. 保持稳定的操作
   - 避免过度转向
   - 避免频繁加减速
   - 保持平滑的动作

2. 覆盖不同的场景
   - 不要总是从同一位置开始
   - 尝试不同的追踪策略
   - 记录不同的失败情况

3. 记录关键信息
   - 采集时间
   - 小球配置
   - 特殊情况（碰撞、搜索等）
```

#### 采集后的验证

```
1. 检查数据完整性
   - 每条轨迹的帧数
   - 状态向量的有效性
   - 动作的多样性

2. 检查数据质量
   - 是否有异常值
   - 是否有缺失数据
   - 是否有重复数据

3. 统计数据分布
   - 距离分布
   - 角度分布
   - 动作分布
```

## 技术指导

### 1. 特征工程的最佳实践

#### 原则 1：特征应该是可解释的

```
✓ 好的特征：
  distance = √((x_ball - x_robot)² + (z_ball - z_robot)²)
  → 清晰、可理解、可验证

✗ 坏的特征：
  feature_42 = 0.3*state[2] + 0.7*state[5]² - 0.2*sin(state[10])
  → 难以理解、难以验证
```

#### 原则 2：特征应该是独立的

```
✓ 好的特征组合：
  [distance, angleDiff, isVisible, isBlocked]
  → 每个特征提供不同的信息

✗ 坏的特征组合：
  [distance, distance², distance³, ...]
  → 特征之间高度相关
```

#### 原则 3：特征应该覆盖决策空间

```
✓ 完整的特征空间：
  - 距离：决定前进速度
  - 角度：决定转向速度
  - 可见性：决定是否搜索
  - 遮挡：决定是否改道
  - 加速度：决定预测能力

✗ 不完整的特征空间：
  - 只有距离和角度
  - 缺少可见性和遮挡信息
  - 无法处理复杂场景
```
