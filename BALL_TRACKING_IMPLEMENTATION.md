## 需求分析

### 项目目标

实现一个基于模仿学习（Imitation Learning）的小车自动跟踪动态小球系统，使用 ACT（Action Chunking with Transformers）框架进行训练。

### 核心需求

1. **小球运动控制**
   - 支持多种运动模式（固定航迹、随机运动、混合运动）
   - 可配置的运动参数（半径、速度、随机强度）
   - 实时计算小球位置和速度

2. **数据采集**
   - 采集人类操作小车跟踪小球的演示数据
   - 记录完整的状态-动作对
   - 包含小球的位置和速度信息

3. **模型训练**
   - 使用采集的数据训练神经网络
   - 学习状态到动作的映射
   - 支持多模态输入（图像 + 状态向量）

4. **自主推理**
   - 训练后的模型自动控制小车
   - 实时跟踪运动的小球
   - 支持不同运动模式的泛化

## 核心问题

### 问题 1：状态表示不足

**原始设计的问题**：
- 只采集小球的距离信息，缺少位置和速度
- 模型无法知道小球在哪个方向
- 模型无法预测小球的运动趋势
- 只能被动反应，无法主动预测

**影响**：
- 模型学习效果差
- 跟踪精度低
- 无法处理快速运动的小球

### 问题 2：角度编码不连续

**原始设计的问题**：
- 使用单个角度值 (0-2π)
- 在 360°→0° 处有不连续跳跃
- 神经网络难以学习这种周期性

**影响**：
- 模型在角度边界处表现不稳定
- 增加训练难度

### 问题 3：网络架构不匹配

**原始设计的问题**：
- 状态输入层只有 32 个神经元
- 无法充分提取 14 维状态的特征
- 网络深度不足

**影响**：
- 特征提取能力弱
- 模型容量不足

## 实现方案

### 方案概述

采用**扩展状态表示 + 改进网络架构**的方案，从根本上解决状态不足的问题。

### 核心改进

#### 1. 小球运动模块

**三种运动模式**：

| 模式 | 实现方式 | 适用场景 |
|------|---------|---------|
| **固定航迹** | 圆形轨迹运动 | 初期调试、基础训练 |
| **随机运动** | 随机游走 | 增加数据多样性 |
| **混合运动** | 圆形轨迹 + 随机扰动 | 平衡可控性和多样性 |

**实现代码**（`updateBallMotion` 函数）：

```typescript
// 固定航迹：圆形运动
const angle = sim.current.ballMotionTime * ballSpeed;
newX = ballRadius * Math.cos(angle);
newZ = ballRadius * Math.sin(angle);

// 随机运动：随机游走
const randomX = (Math.random() - 0.5) * ballRandomIntensity * ballSpeed;
const randomZ = (Math.random() - 0.5) * ballRandomIntensity * ballSpeed;
newX = prevX + randomX;
newZ = prevZ + randomZ;

// 混合运动：固定航迹 + 随机扰动
const baseX = ballRadius * Math.cos(angle);
const baseZ = ballRadius * Math.sin(angle);
const randomX = (Math.random() - 0.5) * ballRandomIntensity * 0.5;
const randomZ = (Math.random() - 0.5) * ballRandomIntensity * 0.5;
newX = baseX + randomX;
newZ = baseZ + randomZ;
```

#### 2. 状态采集扩展

**从 14 维 → 12 维有效状态**

原始状态只有 5 个维度被使用，其余为 0。新方案充分利用所有 12 个维度。

#### 3. 角度编码改进

**使用三角函数编码**：

```typescript
// 原始方式（有不连续性）
state[2] = angle;  // 0 到 2π，在边界处跳跃

// 改进方式（连续）
state[2] = Math.sin(angle);
state[3] = Math.cos(angle);
```

**优势**：
- 避免 360°→0° 的不连续跳跃
- 神经网络更容易学习周期性
- 数值更稳定

## 状态采集设计

### 12 维状态向量详解

```
┌─────────────────────────────────────────────────────┐
│              12 维状态向量结构                        │
├─────────────────────────────────────────────────────┤
│ 维度 0-1:   机器人位置 (x, z)                       │
│ 维度 2-3:   机器人朝向 (sin(θ), cos(θ))             │
│ 维度 4:     机器人速度 (v)                          │
│ 维度 5-6:   小球位置 (ball_x, ball_z)               │
│ 维度 7-8:   小球速度 (ball_vx, ball_vz)             │
│ 维度 9:     目标距离 (distance)                     │
│ 维度 10:    相对角度 (relative_angle)               │
│ 维度 11:    碰撞标志 (is_colliding)                 │
└─────────────────────────────────────────────────────┘
```

### 状态采集流程

```
采集阶段（人类操作）
    ↓
recordFrame() 每 100ms 调用一次
    ↓
构建 12 维状态向量
    ├─ 机器人状态：位置、朝向、速度
    ├─ 小球状态：位置、速度
    ├─ 相对信息：距离、相对角度
    └─ 环境信息：碰撞标志
    ↓
记录人类动作（离散 5 维或连续 2 维）
    ↓
保存 [状态, 动作] 对到当前 Episode
    ↓
采集完成后保存 Episode
```

### 关键计算

**相对角度计算**：

```typescript
// 计算从机器人指向小球的角度
const angleToTarget = Math.atan2(targetX - x, targetZ - z);

// 计算相对于机器人朝向的角度
let relativeAngle = angleToTarget - angle;

// 归一化到 [-π, π]
while (relativeAngle > Math.PI) relativeAngle -= Math.PI * 2;
while (relativeAngle < -Math.PI) relativeAngle += Math.PI * 2;
```

**小球速度计算**：

```typescript
// 每帧计算速度（位置差分）
const dt = 0.016; // ~60fps
sim.current.ballVelX = (newX - prevX) / dt;
sim.current.ballVelZ = (newZ - prevZ) / dt;
```

## 网络架构

### 改进前后对比

**改进前**：
```
输入层：
  - 图像输入：64×64×3
  - 状态输入：14 维（实际只用 5 维）

特征提取：
  - 图像：Conv2d(16) → MaxPool → Conv2d(32) → MaxPool → Flatten
  - 状态：Dense(32)  ← 太小！

融合层：
  - Concatenate([image_features, state_features])
  - Dense(128)
  - Output(20)  ← CHUNK_SIZE=10, 2 个动作
```

**改进后**：
```
输入层：
  - 图像输入：64×64×3
  - 状态输入：12 维（充分利用）

特征提取：
  - 图像：Conv2d(16) → MaxPool → Conv2d(32) → MaxPool → Flatten
  - 状态：Dense(64)  ← 增加到 64！

融合层：
  - Concatenate([image_features, state_features])
  - Dense(128)
  - Dense(64)   ← 添加额外层
  - Output(20)  ← CHUNK_SIZE=10, 2 个动作
```

### 改进的优势

| 方面 | 改进前 | 改进后 | 优势 |
|------|--------|--------|------|
| **状态维度** | 14（5 有效） | 12（12 有效） | 充分利用状态信息 |
| **状态处理** | Dense(32) | Dense(64) | 更强的特征提取 |
| **网络深度** | 2 层 | 3 层 | 更强的非线性能力 |
| **参数量** | ~少 | ~多 | 更大的模型容量 |

## 使用流程

### 完整的工作流程

```
1️⃣ 配置小球运动
   ├─ 打开应用：http://localhost:3000
   ├─ 找到"2. 操控本体"→"小球运动控制"
   ├─ 选择运动模式：固定航迹/随机运动/混合运动
   ├─ 调整参数：
   │  ├─ 半径：1-10 米（推荐 5）
   │  ├─ 速度：0.1-3（推荐 1）
   │  └─ 随机强度：0-2（仅随机/混合模式）
   └─ 点击"复位"重置小车位置

2️⃣ 采集数据（10-20 条轨迹）
   ├─ 点击"开始采集"
   ├─ 手动操作小车跟踪小球
   │  ├─ W/↑：前进
   │  ├─ S/↓：后退
   │  ├─ A/←：左转
   │  └─ D/→：右转
   ├─ 每条轨迹 30-60 秒
   ├─ 采集的状态包含：
   │  ├─ 机器人位置、朝向、速度
   │  ├─ 小球位置、速度
   │  ├─ 相对角度、距离
   │  └─ 碰撞信息
   └─ 采集完成后自动保存

3️⃣ 训练模型
   ├─ 点击"开始训练模型"
   ├─ 模型使用以下输入：
   │  ├─ 图像：64×64 RGB 图像
   │  └─ 状态：12 维状态向量
   ├─ 训练 50 个 Epoch
   ├─ 观察 Loss 下降曲线
   └─ 训练完成后自动保存

4️⃣ 推理验证
   ├─ 点击"启动自主推理"
   ├─ 模型自动控制小车
   ├─ 推理时使用相同的 12 维状态
   ├─ 观察跟踪效果
   ├─ 测试不同的小球运动模式
   └─ 评估跟踪精度
```

```typescript
const updateBallMotion = useCallback(() => {
  if (!sim.current.target) return;

  const dt = 0.016;
  sim.current.ballMotionTime += dt;

  // 根据模式计算新位置
  let newX, newZ;
  if (ballMotionMode === 'fixed') {
    const angle = sim.current.ballMotionTime * ballSpeed;
    newX = ballRadius * Math.cos(angle);
    newZ = ballRadius * Math.sin(angle);
  }
  // ... 其他模式

  // 计算速度
  sim.current.ballVelX = (newX - prevX) / dt;
  sim.current.ballVelZ = (newZ - prevZ) / dt;

  // 更新位置
  sim.current.target.position.x = newX;
  sim.current.target.position.z = newZ;
}, [ballMotionMode, ballRadius, ballSpeed, ballRandomIntensity]);
```
